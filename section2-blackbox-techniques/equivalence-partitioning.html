Equivalence Partitioning

Equivalence partitioning is a black-box test design technique in which input data is divided into partitions. Each partition is expected to behave the same way, and test cases can be designed to test one representative value from each partition.

Real-life Example: Phone Number Validation

Imagine you are testing a form field that accepts phone numbers. The valid phone numbers must be between 100000 and 999999. Use the form below to input different phone numbers and observe how equivalence partitioning can be applied.

Explanation of Equivalence Partitions:

  • Valid Partition: Phone numbers between 100000 and 999999 are valid.
  • Invalid Partition: Phone numbers below 100000 and above 999999 are invalid.
