White-box Testing

White-box testing is a testing technique that analyzes the internal structure and logic of the system being tested. Testers must have knowledge of the code and logic to create test cases that ensure full coverage.

Practical Example: Path Coverage

In the example below, review the code and determine what test cases you would create to ensure 100% path coverage.

function calculateDiscount(price, discount) {
    if (price > 100) {
        return price - (price * discount);
    } else {
        return price;
    }
}

Your Task

Based on the code above, which test cases would you create to ensure that all paths (both the `if` and `else` statements) are covered?

Think about different values for `price` and `discount` that would trigger the `if` and `else` branches.
