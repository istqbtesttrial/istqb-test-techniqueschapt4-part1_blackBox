Overview of Test Techniques

Software testing techniques help in two major activities: test analysis (what to test) and test design (how to test). These techniques allow testers to create systematic, efficient test cases that cover important areas of the system under test.

Types of Test Techniques

  • Black-box Test Techniques: Based on an analysis of the system’s behavior without reference to its internal structure. Examples include equivalence partitioning and boundary value analysis.
  • White-box Test Techniques: Based on an analysis of the internal structure and logic of the system. Examples include path coverage and code analysis.
  • Experience-based Test Techniques: Leverage the knowledge and experience of the tester to find defects. These techniques can catch issues that black-box and white-box techniques might miss.
